Privacy Filter Settings

Runs the openai/privacy-filter model entirely in your browser via Transformers.js. No server or API key needed.
First run downloads the model (~30–50 MB, cached afterwards). WebGPU is used when available, with WASM fallback.
Detects: names, emails, phones, addresses, URLs, dates, account numbers, secrets.

Per-sentence keeps each model call short (useful when inputs are long or the model's context window is limiting); whole-text gives the model full surrounding context, which can improve recall on entities that span or depend on neighboring sentences.

Transformers.js only supports simple and none. Word-level strategies (first / average / max) are Python-transformers only.

Lower threshold → higher recall (more detections, more false positives). Higher threshold → higher precision. 0 disables filtering.

Diagnostics

When off (default), Rescriber writes nothing about your prompts or detected PII to any DevTools console — only errors. Turn this on only if you're actively debugging an issue or asked to by support; it logs prompt/entity metadata (lengths, types, counts) at the verbose level. Turn it back off when you're done.

Stored Data

View and manage all PII mappings, entity counts, action history, and abstract mappings stored locally by Rescriber.
